-------------------------------------

Because Gemma did not support JSON mode or `system_instruction`, assistant updated
`code/agent.py` to:

- Use `models/gemma-3-12b-it`.
- Move system instructions into the user prompt.
- Parse JSON text after stripping optional markdown fences.
- Keep API failures loud.
- Avoid mock or silent fallback behavior.

Assistant added JSON text parsing:

```python
def _parse_json_text(raw_text):
    raw_text = raw_text.strip()
    raw_text = re.sub(r"^```(?:json)?\s*", "", raw_text)
    raw_text = re.sub(r"\s*```$", "", raw_text)
    return json.loads(raw_text.strip())
```

16. Professional Engineering Summary
------------------------------------

The implementation now follows a clean CLI-based RAG architecture:

- Data ingestion is handled by `corpus_fetcher.py`.
- Retrieval is handled locally with deterministic TF-IDF scoring.
- LLM-specific behavior is isolated in `agent.py`.
- `main.py` coordinates input, retrieval, triage, terminal reporting, and output.
- Environment configuration is explicit and fail-fast.
- Unsafe or ambiguous tickets are escalated instead of answered automatically.
- The generated CSV preserves the required schema and contains a balanced mix of
  replied and escalated results.

The final solution avoids silent mock behavior and makes API/configuration errors
visible during execution, which is important for production-quality debugging and
evaluation transparency.
